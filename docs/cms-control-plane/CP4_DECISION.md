# CP4 — Decision

## 1. Endelig beslutning

**GO WITH CONDITIONS**

## 2. Hva som er oppnådd

- CMS er **tydeligere hovedbase** for kontroll: `CONTROL_PLANE_DOMAIN_ACTION_SURFACES` + kort på **domener** og **control**.
- **Domener:** firma/kunder, avtale-runtime (plan/binding-innsyn), uke/meny governance-panel, tårn (admin/kitchen/driver/superadmin), growth (SEO/social/ESG) — alle med **kilde, posture og handlinger**.
- **Ukemeny:** operativ kjede forblir `GET /api/week` + avtale + Sanity meny; `weekPlan` **editorial-only** med amber UI; **Published**-etikett for LIVE i strip.
- **Control towers:** forklart som **runtime_route** via action surfaces, ikke bare løse lenker.

## 3. Hva som fortsatt er svakt

- Worker **STUB**, social **DRY_RUN**, SEO/ESG **LIMITED** (uendret ærlighet).
- To fortellinger (operativ meny vs editorial weekPlan) — **eksplisitte i UI**, men fortsatt produktmessig kompleksitet.

## 4. Umbraco-/verdensklasse-nivå

**Ærlig:** Sterk governance- og navigasjonskonsistens i backoffice; ikke full multi-site editor-plattform med universelle workflows.

## 5. Før ubetinget enterprise-live-ready

1. **Worker** — reell implementasjon eller isolasjon.
2. **Audit/hardening** (middleware/API) — lukk gule/røde funn.
3. **Observability** — etter roadmap.

## 6. Kan vente

- Dypere Studio deep-links (når URL-struktur er stabil).
- Ekstra polish på action-kort uten semantikkendring.
