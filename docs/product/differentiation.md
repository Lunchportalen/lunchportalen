# Differensiering — Lunchportalen

Sammenligningen er **prinsipiell** — ikke en funksjonsliste mot konkrete konkurrenter. Bruk den til å styre samtalen med kunder og investorer.

## Mot byråer (marketing / performance)

| Byrå | Lunchportalen |
|------|----------------|
| Ekstern eierskap til kontoer og rapporter | **Eierdata og prosess** ligger hos kunden i plattformen |
| Ofte prosjekt- eller retainerbasert | **SaaS** med forutsigbar kost og kontinuerlig produkt |
| Varierende grad av operasjonell forståelse | **Vertikal dominans** — bygget rundt mat, ordre og drift |

**Poeng:** Byrå kan være **partner**; Lunchportalen er **systemet** byrået eller kunden jobber innenfor.

## Mot rene annonseverktøy

| Ad-verktøy | Lunchportalen |
|------------|----------------|
| Optimalisering innenfor annonsesjanger | **Kobling til ordre, margin og intern policy** der implementert |
| Fokus på klikk og konverteringshendelser i plattformen | Fokus på **forretningsutfall** og **kontrollert spend** |
| Lite naturlig sted for kjøkken/bestilling | **Operativ kjerne** er innebygd |

**Poeng:** Annonseverktøy svarer på «hvordan annonsere»; Lunchportalen adresserer **«hvordan vi driver matvirksomhet lønnsomt»**.

## Mot rene analyse- og BI-verktøy

| Analyse/BI | Lunchportalen |
|------------|----------------|
| Visualiserer data du mater inn | **Produserer og strukturerer** operative og vekstdata i samme flyt |
| Krever ofte tung integrasjon for handling | **Beslutning og handling** kan skje i samme system (med kontroll) |
| Sjeldent vertikal for mat | **Domene-modell** for bedriftsmat og relaterte roller |

**Poeng:** BI er **glassrute**; Lunchportalen er **cockpit** (med begrensninger — ikke overdrive).

## Kort «elevator»-differensiering

«Vi er ikke bare annonser eller bare analyse. Vi er **operativ plattform for bedriftsmat** med **AI og inntektslogikk inne i flyten** — med **roller og sporbarhet** enterprise forventer.»
