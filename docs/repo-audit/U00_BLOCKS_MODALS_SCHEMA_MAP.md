## Schema / UI / Preset Split
| Layer | Real files | What it is |
|---|---|---|
| Schema truth | `blockFieldSchemas.ts`, `contentDocumentTypes.ts`, `bodyEnvelopeContract.ts`, `blockAllowlistGovernance.ts` | Static TypeScript contract and validation logic. |
| Configured instance | `content_page_variants.body`, envelope fields, selected `documentType` | Saved page/block state in the database. |
| UI surfaces | `BlockAddModal`, `BlockPickerOverlay`, `BlockEditModal`, `SchemaDrivenBlockForm`, `RightPanel`, `ContentWorkspaceMainCanvas` | What editors actually interact with. |
| Presets / registries | `editorBlockCreateOptions.ts`, `blockRegistry.ts`, `backofficeSchemaSettingsModel.ts` | Descriptions and available create options presented to users. |

## System Judgment
This editor has real block/schema/governance machinery. The weakness is not absence. The weakness is split ownership: create options, block registry, field schemas, document types, preview surfaces, and inspector scope all live in different files and mental models, while the user experiences them as one workspace.
