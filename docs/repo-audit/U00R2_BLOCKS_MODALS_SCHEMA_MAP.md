## Layer Split
| Layer | Exact files | What it really is |
|---|---|---|
| Schema | `blockFieldSchemas.ts`, `contentDocumentTypes.ts`, `blockAllowlistGovernance.ts` | Static governance truth in TypeScript. |
| Configured instance | `content_page_variants.body`, `bodyEnvelopeContract.ts`, `legacyEnvelopeGovernance.ts` | Saved content payload and migration bridge. |
| UI | `BlockAddModal.tsx`, `BlockPickerOverlay.tsx`, `BlockEditModal.tsx`, `SchemaDrivenBlockForm.tsx`, `RightPanel.tsx` | What editors actually interact with. |
| Presets / registries | `editorBlockCreateOptions.ts`, `blockRegistry.ts`, `designContract.ts`, `backofficeSchemaSettingsModel.ts` | Descriptions and defaults presented to users. |

## System Judgment
This editor has real block/schema/governance machinery. The weakness is split ownership: create options, block registry, field schemas, document types, preview surfaces, global settings, and inspector scope all live in different files and mental models, while the user experiences them as one workspace.
