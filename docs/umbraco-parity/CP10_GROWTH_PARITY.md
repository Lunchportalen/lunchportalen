# CP10 — Growth parity (WS6)

## Social / SEO / ESG

- **CP10** endrer ikke growth-backend eller modulspesifikk logikk.
- **Krav:** modulstatus (LIVE / LIMITED / DRY_RUN / STUB / INTERNAL_ONLY) skal **ikke** kunne mistolkes som full produksjon der backend er begrenset.

## Navigasjon

- `SEO`, `Social`, `ESG` inkludert i `BACKOFFICE_NAV_ITEMS` og dermed i palett.

## Verifikasjon

- Manuelt: palett → «social» → `/backoffice/social`.
