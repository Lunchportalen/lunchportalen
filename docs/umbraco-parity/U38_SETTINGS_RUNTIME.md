# U38 Settings Runtime

## Landed

- Settings tabs now come from the canonical settings collection registry.
- Document type and data type detail views moved from passive tables toward management-flow cards.
- Settings and editor governance now reference the same property-editor/system model.

## Result

- Settings is more operational and less essay-like.
- Object relationships are visible at the point of use.
- The section hierarchy is closer to Bellissima’s collection/workspace mental model.

## Remaining Gap

- Some management surfaces are still read-mostly because persisted CRUD does not yet exist for those objects.
- U38 does not claim full parity without the missing route screenshots.
