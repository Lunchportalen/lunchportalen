# CP9 — Decision

## 1. Endelig beslutning

**GO WITH CONDITIONS**

- **GO:** CP9 leverer **editorial UX-løft** (innholdstre-søk) + full dokumentasjonskontrakt for workflow/history/domain.
- **Betingelser:** «Null merkbare gap» for **alle** brukere krever brukertest og videre IA; **global søk** og **unified history** krever mer enn CP9.

## 2. Hva som er oppnådd

- **CMS** som hovedbase uendret i sannhet; **redaktør** får raskere navigasjon i **content tree**.
- **Domener** fortsatt via surfaces + dokumentasjon.
- **Uke/meny** uendret i runtime; publish fortsatt Sanity + CP7 broker der satt.
- **Tårn** fortsatt innordnet via kontrollplan.

## 3. Hva som fortsatt er svakt

- Global backoffice-søk.
- Én tidslinje for historikk på tvers av Postgres og Sanity.

## 4. Nærhet til Umbraco-/verdensklasse

- **Tree UX:** nærmere etter CP9.
- **Full plattform-paritet:** krever replatforming eller flere iterasjoner.

## 5. Før ubetinget enterprise-live-ready

1. Brukertest på backoffice (content vs meny).
2. Vurder lenke-palett (⌘K) uten ny indeks.
3. Runbook for publish-token.

## 6. Kan vente

- Server-side fulltext, unified history UI.
