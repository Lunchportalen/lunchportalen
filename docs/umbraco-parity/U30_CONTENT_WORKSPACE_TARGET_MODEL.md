# U30 — Content workspace target model

## Soner (venstre → høyre)

1. **Section tree (ytterst):** Bred nok til skanning; søk øverst; tydelig feilbanner ved API-nekt/migrasjon.
2. **Struktur (tri-pane venstre):** Blokk-outline — sekundær navigasjon.
3. **Editor + preview (midt):** Én sammenhengende arbeidsflate; preview i split skal ha **minst** ~440px effektiv kolonne og visuell vekt.
4. **Inspector (høyre):** Content apps (Egenskaper / AI / SEO / …) — stor nok touch/lesing; egenskaps-innhold gruppert under «Innholdsapper».

## Tone ned

- Duplikat utkast/publisert/sync-linjer i **både** topbar og editor-stripe.
- Overflødige små `rounded-full` badges i samme skjermhøyde.

## Fremhev

- Publiser + lagre (allerede forsterket i U29R — behold én sannhet i topp).
- Preview-kolonne (bredde + rolig bakgrunn).

## Primær vs sekundær

- **Primær:** Lagre, Publiser, rediger/forhåndsvisning toggle.
- **Sekundær:** Kopier support, last på nytt, enhetsvalg, offentlig URL.
