# U38 Property Editor Runtime

## Landed

- Added a canonical flow model for document-type-driven field governance.
- Added a canonical flow model for property-editor kinds/data types.
- Surfaced the same flow in:
  - settings document type detail
  - settings data type detail
  - content workspace governance rail

## Visible Model

- Schema: field kind and contract
- Configured instance: where that kind is used
- UI: which editor mapping renders it
- Preset/defaults: which defaults and presets shape the value start state

## Outcome

- “What drives this field?” is now answerable from the editor.
- “Where is this kind used?” is now answerable from settings.
